# 账号管理模块快速测试指南 🚀

## 📋 测试前准备

### 1. 启动后端服务
```bash
cd backend
mvn spring-boot:run
```
等待看到 "Started VolunteerApplication" 提示。

### 2. 启动前端服务
```bash
cd frontend
npm run dev
```
访问 http://localhost:5173

### 3. 登录管理员账号
- 角色选择：**管理员**
- 账号：`admin`
- 密码：`admin123`（根据实际配置）

---

## 🧪 功能测试步骤

### 测试 1：访问账号管理模块

1. ✅ 登录成功后，查看左侧菜单
2. ✅ 找到"账号管理维护"菜单项（带 Users 图标）
3. ✅ 点击进入，页面应显示两个 Tab：
   - 学生账号管理
   - 负责人账号管理

---

### 测试 2：学生账号管理

#### 2.1 查看学生列表
- ✅ 默认显示"学生账号管理" Tab
- ✅ 表格显示：学号、姓名、学院、联系电话、操作
- ✅ 底部显示分页信息

#### 2.2 搜索功能
1. 在搜索框输入学生姓名或学号
2. 点击"搜索"按钮
3. ✅ 表格应只显示匹配的结果
4. 点击"重置"按钮
5. ✅ 恢复显示所有数据

#### 2.3 新增学生
1. 点击"+ 新增用户"按钮
2. 弹窗标题应为"新增学生账号"
3. 填写表单：
   ```
   学号: 2025001
   姓名: 测试学生
   学院: 计算机学院
   联系电话: 13800138000
   初始密码: (留空，使用默认)
   ```
4. 点击"确定"
5. ✅ 应提示"学生账号创建成功，初始密码：123456"
6. ✅ 列表自动刷新，新学生出现在列表中

#### 2.4 测试学号查重
1. 再次点击"+ 新增用户"
2. 输入相同的学号 `2025001`
3. 点击"确定"
4. ✅ 应提示"学号已存在"

#### 2.5 编辑学生
1. 找到刚创建的学生，点击"编辑"按钮
2. 弹窗标题应为"编辑学生账号"
3. ✅ 学号字段应为禁用状态（灰色）
4. 修改姓名为"测试学生-已修改"
5. 点击"确定"
6. ✅ 应提示"学生信息更新成功"
7. ✅ 列表中姓名已更新

#### 2.6 重置密码
1. 点击该学生的"重置密码"按钮
2. ✅ 弹出确认框："确定要重置 测试学生-已修改 的密码吗？密码将重置为：123456"
3. 点击"确定"
4. ✅ 应提示"密码已重置为：123456"

#### 2.7 删除学生
1. 点击该学生的"删除"按钮
2. ✅ 弹出确认框："确定要删除 测试学生-已修改 的账号吗？此操作不可恢复！"
3. 点击"确定"
4. ✅ 应提示"学生账号删除成功"
5. ✅ 该学生从列表中消失

#### 2.8 分页测试
1. 如果数据超过 10 条，测试分页功能
2. ✅ 点击"下一页"，应显示第 2 页数据
3. ✅ 点击"上一页"，应返回第 1 页
4. ✅ 第 1 页时"上一页"按钮应禁用
5. ✅ 最后一页时"下一页"按钮应禁用

---

### 测试 3：负责人账号管理

#### 3.1 切换到负责人 Tab
1. 点击"负责人账号管理" Tab
2. ✅ 表格列应变为：工号、姓名、部门、联系电话、操作

#### 3.2 新增负责人
1. 点击"+ 新增用户"按钮
2. 弹窗标题应为"新增负责人账号"
3. 填写表单：
   ```
   工号: HEAD001
   姓名: 测试负责人
   部门: 国志协
   联系电话: 13900139000
   初始密码: (留空)
   ```
4. 点击"确定"
5. ✅ 应提示"负责人账号创建成功，初始密码：123456"

#### 3.3 测试工号查重
1. 尝试添加相同工号 `HEAD001`
2. ✅ 应提示"工号已存在"

#### 3.4 编辑负责人
1. 点击"编辑"按钮
2. ✅ 工号字段应禁用
3. 修改部门为"校团委"
4. ✅ 应提示"负责人信息更新成功"

#### 3.5 重置负责人密码
1. 点击"重置密码"
2. ✅ 确认后提示"密码已重置为：123456"

#### 3.6 删除负责人
1. 点击"删除"
2. ✅ 确认后提示"负责人账号删除成功"

---

## 🔍 边界情况测试

### 测试 4：表单验证

#### 4.1 必填字段验证
1. 点击"+ 新增用户"
2. 不填写任何字段，直接点击"确定"
3. ✅ 浏览器应阻止提交，提示"请填写此字段"

#### 4.2 学号格式验证
1. 新增学生时，输入非数字学号（如 "ABC123"）
2. 点击"确定"
3. ✅ 应提示"学号必须为数字"

#### 4.3 空搜索
1. 搜索框留空，点击"搜索"
2. ✅ 应显示所有数据（不过滤）

#### 4.4 搜索无结果
1. 搜索一个不存在的姓名（如 "不存在的人"）
2. ✅ 表格应显示"暂无数据"

## 🎯 性能测试

### 测试 5：大数据量测试

1. 如果数据库中有大量数据（>100 条）
2. ✅ 分页加载应流畅，无明显卡顿
3. ✅ 搜索响应应在 1 秒内
4. ✅ 切换 Tab 应立即响应

## 🐛 错误处理测试

### 测试 6：网络异常

#### 6.1 后端未启动
1. 停止后端服务
2. 尝试加载账号列表
3. ✅ 应提示"加载数据失败，请稍后重试"

#### 6.2 操作失败
1. 停止后端服务
2. 尝试新增账号
3. ✅ 应提示"操作失败，请稍后重试"
